\section*{Abstract}
\addcontentsline{toc}{section}{Abstract}

This project addresses privacy in online sealed-bid auctions, where current
systems require either a trusted auctioneer who sees all bids or a public
ledger that exposes them.  The central question is whether a decentralised
auction can protect both bid values and participant network identities without
any trusted intermediary.  Prior work on MPC-based auctions has relied on
small sets of known, trusted servers communicating over direct TCP
connections, leaving the combination of multi-party computation with
transport-layer anonymity unexplored.  This project combined the MASCOT
protocol (which computes the auction winner without revealing losing bids,
even if all but one party is corrupt) with Veilid, a peer-to-peer framework
whose onion-routed private routes hide participants' IP addresses from each
other and from network observers.  A working prototype was implemented in
Rust (${\sim}$15,000 lines), using Veilid's distributed hash table as the
sole coordination substrate, and benchmarked across devnets of 40--80 nodes
with both the MASCOT and Shamir protocols under party counts from 3 to 20.
The key finding is that MPC over a privacy-preserving overlay is feasible:
the tunnel sustained hundreds of megabytes of MASCOT traffic and produced
correct results at up to 12 parties, with the 15--19$\times$ overhead from
onion routing representing the quantifiable cost of hiding participants'
network identities. This cost that is fundamental to the privacy model but
acceptable for the small party counts typical of real auctions.
